## 🧠 Strategic Principles (The "Fence")
1. **Architecture First**: Never write code without a clear path (STATUS.md).
2. **Determinism**: Use "Style Cloners" (Python scripts with hardcoded values) over "Vibe Coding" (LLM guesses).
3. **Resources**: 
   - Never burn paid API credits for testing (Mock first).
   - Use `token.json` for persistent Auth (never ask user for manual login).
   - "Golden 10": Establish 10 perfect examples manually, then scale.
4. **Source of Truth**: Never re-engineer or "mock" master assets if a User-provided sheet exists. Use exactly what the User provides in `STATUS.md` or `golden_assets.json`.

## 📂 File Structure Standards
**The "Pioneer Pattern" (Root-Level Context):**
- `/AGENTS.md`: THIS file. The immutable Constitution. (Read First).
- `/STATUS.md`: The dynamic Dashboard. Current Phase, Objectives, and Next Steps. (Read Second).
- `/MEMORY.md`: The Long-Term Brain. Strategic decisions, failure logs, and architectural patterns. (Read Third).
- `/scripts/`: All executable logic (Python).
- `/templates/`: Static assets (CSVs, Docs).

## 🔨 key Commands
- **Auth Repair**: `python scripts/refresh_google_oauth.py`
- **Style Check**: `python scripts/rebuild_golden_sheet_bulletproof.py`
- **Lead Gen**: `python scripts/batch_generate_assets.py`

## 🏁 Session End Protocol
1. **Update Context**: Refresh `STATUS.md` and `MEMORY.md`.
2. **Update Artifacts**: Mark progress in `task.md` / `implementation_plan.md`.
3. **Archive**: Move unused scripts/logs to `_archive/`.
4. **Sync Context**: Run `python scripts/sync_context_to_doc.py` to update Google Doc.
5. **Prune**: Verify `MEMORY.md` < 100 lines. (Prevent Context Bloat).
6. **GitHub Sync**: `git add .`, `git commit -m "..."`, `git push`. (Run sequentially).

## 🚫 Anti-Patterns (Do NOT Do)
- **Deep Nesting**: Do NOT create `.agent/plans/specs/v1/...`. Keep it flat.
- **Ambiguous Styling**: Do NOT assume "Default formatting". Use Hex Codes (#FFFFFF), Pixel Heights (40px), and Alignment (Center).
- **Ghost Changes**: Do NOT edit files without verifying the path exists.

## 🎨 Golden 10 Style Spec
**Sheet**:
- **Font**: Arial/Inter, Size 10.
- **Header**: Row 1 = Column Names. Dark Grey BG (#434343), White Text, Bold. Frozen.
- **Intro**: NONE. (No text block above headers).
- **Rows**: 40px Height (Breathing Room).
- **Cols**: "Company" (250px), "Email" (200px).
- **Text**: Vertically Centered, Wrapped.

**Doc**:
- **Margin**: 1 inch.
- **Title**: H1 (Arial 24, Bold).
- **Body**: Arial 11, 1.15 spacing.
- **Intro**: "Executive Brief" style.

## 1. Strategic Principles (The Playbook)

### Resource Efficiency
1. **Never burn paid API credits for testing.** Mock first.

### Architecture & Code

2. **Constants over magic numbers.** Define at top of file.
3. **Trace dependencies.** `grep` before changing ANY value.

## 3. Critical Failure Learnings

- **[Batch Jobs]**: For long-running jobs, use `modal deploy` + spawn async.
- **[Row Heights]**: V5 used 350px (WRONG). Golden spec is 275px.
- **[Token Scopes]**: Not all OAuth tokens have `docs` scope. Check before using.
- **[Google Emails]**: Instantly rejects Gmail addresses. Filter before campaign.
- **[Link Mismatches]**: GDoc links in emails must match Lead Magnet column.
- **[OAuth Clients]**: Use "Desktop client Antigravity" credentials (client ID `921220435908-...`). Old "web" client causes `invalid_scope` errors. Move consent screen to **Production** mode to avoid 7-day testing expiry.
